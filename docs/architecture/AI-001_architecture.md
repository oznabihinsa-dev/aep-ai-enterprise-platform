# 1. Назначение документа

## 1.1. Что такое AI-ОФИС

AI-ОФИС — это цифровая организационная система, предназначенная для выполнения рабочих задач пользователя с применением специализированных AI-сотрудников, единой маршрутизации задач, стандартов обработки информации и централизованного хранения документов.

AI-ОФИС не является набором отдельных чатов. Он рассматривается как цифровая организация, в которой каждый AI-сотрудник имеет определенную роль, зону ответственности, входные данные, ожидаемый результат и ограничения.

AI-ОФИС v1.0 создается для сопровождения рабочих процессов по следующим компаниям:

* ООО «ТЕПЛО»;
* ООО «ВЭЛС»;
* ООО «ДВТК»;
* ООО «КАСКАД».

## 1.2. Для чего создается AI-ОФИС v1.0

AI-ОФИС v1.0 создается для упорядочивания рабочих задач, документов, расчетов и коммуникаций, связанных с деятельностью указанных компаний.

Основное назначение версии v1.0 — сформировать базовую архитектуру цифровой организации, в которой:

* каждая задача проходит через понятный маршрут;
* для каждой задачи определяется ответственный AI-сотрудник;
* исходные документы хранятся в Google Drive;
* ChatGPT Projects используются как рабочие кабинеты AI-сотрудников;
* результаты работы оформляются по единым стандартам;
* риски проверяются до выдачи результата пользователю.

## 1.3. Какие компании входят в контур AI-ОФИС

В контур AI-ОФИС v1.0 входят следующие компании:

### ООО «ТЕПЛО»

Компания используется в задачах, связанных с коммерческими предложениями, поставками угля, расчетами стоимости, договорами, закупками, письмами, претензиями и сопроводительной документацией.

### ООО «ВЭЛС»

Компания используется в задачах, связанных с коммерческой, договорной, закупочной, расчетной и деловой документацией.

### ООО «ДВТК»

Компания используется в задачах, связанных с коммерческими предложениями, поставками, документацией по качеству, деловой перепиской, договорами и расчетами.

### ООО «КАСКАД»

Компания используется в задачах, связанных с перевозками, транспортными услугами, претензионной работой, договорами, актами, расчетами и деловой перепиской.

## 1.4. Границы версии v1.0

AI-ОФИС v1.0 охватывает базовую организационную и документальную архитектуру.

В v1.0 входят:

* структура AI-ОФИС как цифровой организации;
* перечень первых AI-сотрудников;
* правила маршрутизации задач;
* структура проектов ChatGPT;
* правила работы с Google Drive;
* правила именования чатов и файлов;
* базовые стандарты подготовки документов;
* план внедрения AI-ОФИС v1.0;
* журнал изменений архитектуры.

В v1.0 не предполагается создание полноценной внешней информационной системы, отдельного программного продукта, базы данных или автоматической интеграции между всеми сервисами.

## 1.5. Что не входит в v1.0

В версию v1.0 не входят:

* разработка отдельного программного обеспечения AI-ОФИС;
* автоматическая синхронизация ChatGPT Projects с Google Drive;
* автоматическая проверка всех файлов без участия пользователя;
* самостоятельная отправка документов контрагентам без команды пользователя;
* автоматическое подписание документов;
* принятие юридически значимых решений без проверки пользователем;
* замена бухгалтера, юриста, руководителя или ответственного специалиста;
* хранение исходных документов только внутри ChatGPT Projects.

AI-ОФИС v1.0 является управленческо-документальной архитектурой. Его задача — организовать работу AI-сотрудников, а не заменить внутренние системы учета, электронный документооборот или юридическую экспертизу.

# 2. Цели системы AI-ОФИС

## 2.1. Главная цель AI-ОФИС

Главная цель AI-ОФИС — создать управляемую цифровую организацию для обработки рабочих задач, документов, расчетов и переписки по компаниям ООО «ТЕПЛО», ООО «ВЭЛС», ООО «ДВТК» и ООО «КАСКАД».

AI-ОФИС должен обеспечить единый порядок работы: от поступления задачи до выдачи результата и сохранения итоговых документов в Google Drive.

## 2.2. Управленческие цели

Управленческие цели AI-ОФИС:

1. Разделить рабочие задачи по направлениям.
2. Назначить для каждого направления отдельного AI-сотрудника.
3. Исключить хаотичное выполнение задач в случайных чатах.
4. Обеспечить единый вход задач через Главного диспетчера.
5. Закрепить роль Архитектора системы для развития и контроля AI-ОФИС.
6. Создать понятную схему взаимодействия между пользователем, Главным диспетчером, AI-сотрудниками и Google Drive.
7. Обеспечить управляемое развитие AI-ОФИС по версиям.

## 2.3. Операционные цели

Операционные цели AI-ОФИС:

1. Ускорить подготовку коммерческих предложений, писем, претензий, договорных материалов, закупочных документов и расчетов.
2. Сократить количество повторных действий при подготовке типовых документов.
3. Обеспечить быстрый выбор нужного AI-сотрудника для каждой задачи.
4. Установить единые маршруты для типовых задач.
5. Снизить зависимость результата от того, в каком чате была начата работа.
6. Обеспечить повторяемость результата при выполнении похожих задач.
7. Упростить контроль статуса задачи и состава необходимых данных.

## 2.4. Документальные цели

Документальные цели AI-ОФИС:

1. Закрепить Google Drive как основной архив исходных и итоговых документов.
2. Разделить хранение документов по компаниям, направлениям и типам задач.
3. Установить правила именования файлов.
4. Установить правила поиска документов перед началом работы.
5. Установить правила версионности документов.
6. Исключить ситуацию, когда важный документ остается только в чате.
7. Обеспечить возможность повторного использования шаблонов, расчетов и ранее подготовленных материалов.

## 2.5. Цели по снижению ошибок и потерь

AI-ОФИС должен снижать следующие риски:

1. Использование неверной компании в документе.
2. Ошибки в реквизитах.
3. Использование устаревшего файла или неактуальной редакции документа.
4. Неверный расчет цены, НДС, доставки, перевалки или иных составляющих стоимости.
5. Смешение документов разных компаний.
6. Подготовка ответа без проверки исходного запроса.
7. Потеря результата из-за отсутствия сохранения в Google Drive.
8. Неправильный выбор AI-сотрудника.
9. Подготовка юридически значимого документа без фиксации рисков.

## 2.6. Цели по ускорению работы

AI-ОФИС должен ускорить работу за счет:

1. Единой классификации входящих задач.
2. Готовых маршрутов для типовых задач.
3. Использования специализированных AI-сотрудников.
4. Накопления стандартов, шаблонов и правил.
5. Понятной структуры ChatGPT Projects.
6. Понятной структуры Google Drive.
7. Повторного использования проверенных формулировок и расчетных подходов.
8. Минимизации лишних уточнений, если данные уже есть в задаче или в архиве.

# 3. Принципы архитектуры AI-ОФИС

## 3.1. AI-ОФИС как цифровая организация, а не набор чатов

AI-ОФИС строится как цифровая организация с распределением ролей и ответственности.

Каждый чат, проект и AI-сотрудник должны иметь назначение. Нельзя создавать AI-сотрудника без определения:

* роли;
* задач;
* входных данных;
* результата;
* рисков;
* ограничений;
* места в общей структуре AI-ОФИС.

Чаты внутри проектов не должны использоваться хаотично. Каждый чат должен относиться к конкретной задаче, компании, контрагенту, объекту или направлению работы.

## 3.2. Разделение ролей AI-сотрудников

В AI-ОФИС применяется принцип специализации.

Каждый AI-сотрудник отвечает за отдельное направление работы. Это нужно для повышения качества результата, сохранения контекста и исключения смешения задач.

Примеры специализации:

* Главный диспетчер — принимает и классифицирует задачи;
* AI-сотрудник по коммерческим предложениям — готовит КП и ответы на запросы цен;
* AI-сотрудник по закупкам и тендерам — анализирует закупочную документацию и готовит материалы заявки;
* AI-сотрудник по договорам и претензиям — проверяет договоры, готовит претензии и юридически аккуратную переписку;
* AI-сотрудник по Excel — выполняет расчеты, таблицы и печатные формы;
* AI-сотрудник по письмам — готовит деловую переписку;
* AI-сотрудник по IT — сопровождает задачи по Java, Docker, PostgreSQL, серверам и техническим вопросам;
* AI-сотрудник по Google Drive и архиву — отвечает за правила хранения и поиска документов;
* AI-сотрудник по аналитике и поиску информации — выполняет внешние информационные проверки.

## 3.3. Единый вход задач через Главного диспетчера

Входящая рабочая задача должна сначала проходить через Главного диспетчера AI-ОФИС.

Главный диспетчер определяет:

1. Что нужно сделать.
2. К какой компании относится задача.
3. Какой тип задачи поступил.
4. Какие данные нужны.
5. Какой AI-сотрудник должен выполнять задачу.
6. Какой результат должен быть на выходе.
7. Какие риски необходимо проверить.

Главный диспетчер не должен выполнять узкую работу полностью, если задача относится к зоне ответственности специализированного AI-сотрудника. Его основная функция — первичный разбор, маршрутизация и контроль правильного выбора исполнителя.

## 3.4. Google Drive как основной архив документов

Google Drive является основным архивом документов AI-ОФИС.

В Google Drive должны храниться:

* исходные документы;
* запросы заказчиков;
* договоры;
* приложения;
* спецификации;
* счета;
* акты;
* накладные;
* сертификаты качества;
* протоколы испытаний;
* коммерческие предложения;
* претензии;
* письма;
* расчеты;
* шаблоны;
* итоговые документы.

ChatGPT Projects не являются архивом документов. Они используются для выполнения задач, хранения рабочего контекста и взаимодействия с AI-сотрудниками. Все значимые исходные и итоговые документы должны быть сохранены в Google Drive.

## 3.5. ChatGPT Projects как рабочие кабинеты AI-сотрудников

ChatGPT Projects используются как рабочие кабинеты AI-сотрудников.

Каждый проект должен соответствовать определенному направлению работы. Внутри проекта допускается создавать отдельные чаты по конкретным задачам.

Проект не должен подменять архив документов. Его задача — обеспечить рабочую среду для AI-сотрудника, а не долгосрочное хранение исходных файлов.

Для каждого проекта должны быть определены:

* назначение проекта;
* типы задач;
* правила именования чатов;
* используемые документы;
* ожидаемые результаты;
* ограничения;
* связь с папками Google Drive.

## 3.6. Документируемость всех решений

Решения, правила и стандарты AI-ОФИС должны фиксироваться в документах.

Ключевые элементы архитектуры не должны существовать только в переписке. Если правило используется постоянно, оно должно быть включено в соответствующий документ AI-ОФИС.

Документированию подлежат:

* архитектура AI-ОФИС;
* перечень AI-сотрудников;
* карточки AI-сотрудников;
* маршруты типовых задач;
* правила хранения документов;
* правила именования файлов и чатов;
* шаблоны документов;
* журнал изменений;
* стандарты проверки результата.

## 3.7. Проверяемость исходных данных

Перед подготовкой результата AI-сотрудник должен определить, какие исходные данные использованы.

Если данных недостаточно, AI-сотрудник должен указать, чего именно не хватает. Если данные есть в Google Drive, необходимо выполнить поиск по установленным правилам.

Для задач с расчетами, договорами, закупками, претензиями и коммерческими предложениями необходимо проверять:

* компанию;
* контрагента;
* предмет задачи;
* дату;
* номер договора, закупки, счета или иного документа;
* актуальность файла;
* применимость шаблона;
* наличие НДС или основания его отсутствия;
* исходные числовые значения;
* документы, подтверждающие расчет или вывод.

## 3.8. Контроль рисков перед выдачей результата

Перед выдачей результата AI-сотрудник должен определить риски, связанные с задачей.

Риски могут быть:

* юридические;
* финансовые;
* налоговые;
* расчетные;
* закупочные;
* документальные;
* организационные;
* технические.

Если риск существенный, он должен быть явно указан пользователю. AI-сотрудник не должен скрывать неопределенность, отсутствие данных или необходимость дополнительной проверки.

## 3.9. Единые стандарты оформления документов

AI-ОФИС должен использовать единые стандарты оформления документов.

Документы должны быть:

* структурированными;
* пригодными для дальнейшего редактирования;
* оформленными в деловом стиле;
* без лишних формулировок;
* с понятной логикой;
* с корректным указанием компании, адресата, даты, предмета и приложений;
* с расчетами, которые можно проверить.

Для повторяющихся задач должны использоваться шаблоны.

## 3.10. Постепенное внедрение через v1.0 → v1.1 → v2.0

AI-ОФИС внедряется поэтапно.

Версия v1.0 предназначена для создания базовой архитектуры, первых AI-сотрудников, правил хранения документов и маршрутов типовых задач.

Версия v1.1 может включать уточнение ролей, расширение перечня AI-сотрудников, улучшение шаблонов, настройку дополнительных маршрутов и корректировку структуры Google Drive.

Версия v2.0 может включать более сложные сценарии автоматизации, интеграции, расширенные регламенты, базы знаний, контрольные панели и другие инструменты управления AI-ОФИС.
