# BP-001 — Подготовка коммерческого предложения

Код: BP-001  
Версия: 0.1-alpha  
Статус: draft  
Дата: 2026-06-29  
Связанные документы: AI-001, REG-001, STD-001  

## 1. Назначение процесса

Процесс BP-001 описывает порядок подготовки коммерческого предложения по товарам и услугам для компаний:

- ООО «ТЕПЛО»;
- ООО «ВЭЛС»;
- ООО «ДВТК»;
- ООО «КАСКАД».

## 2. Участники процесса

| Роль | Ответственность |
|---|---|
| Пользователь | Ставит задачу и утверждает результат |
| Главный диспетчер | Классифицирует задачу и выбирает исполнителя |
| Менеджер коммерческих предложений | Готовит КП |
| Специалист по Excel | Проверяет расчет, если он сложный |
| Юрист | Проверяет риски, если КП связано с договором или спором |
| Google Drive | Хранит исходные и итоговые документы |

## 3. Входные данные

Для подготовки КП нужны:

1. Компания, от имени которой готовится КП.
2. Заказчик.
3. Запрос заказчика.
4. Вид товара или услуги.
5. Марка, фракция, характеристики.
6. Объем.
7. Место поставки.
8. Условия доставки.
9. Цена или файл калькуляции.
10. Ставка НДС.
11. Срок действия КП.
12. Бланк организации.
13. Реквизиты организации.
14. Сертификат качества или иной подтверждающий документ.

## 4. Маршрут процесса

1. Пользователь передает задачу Главному диспетчеру.
2. Главный диспетчер определяет компанию и тип задачи.
3. Главный диспетчер направляет задачу AI-сотруднику по КП.
4. AI-сотрудник по КП определяет недостающие данные.
5. AI-сотрудник по КП ищет нужные документы в Google Drive.
6. При необходимости подключается AI-сотрудник по Excel.
7. Выполняется расчет цены.
8. Проверяются сертификаты и характеристики.
9. Готовится текст коммерческого предложения.
10. Выполняется контрольный чек-лист.
11. Результат передается пользователю.
12. Итоговый документ сохраняется в Google Drive.

## 5. Контрольный чек-лист

Перед выдачей КП проверить:

- указана правильная компания;
- указан заказчик;
- цена подтверждена калькуляцией или задана пользователем;
- НДС рассчитан корректно;
- характеристики подтверждены сертификатом;
- ГОСТ указан только при подтверждении источником;
- адрес и реквизиты заказчика не взяты из старого образца без проверки;
- срок действия КП указан;
- приложения перечислены;
- документ не содержит пустых полей.

## 6. Риски

| Риск | Проверка |
|---|---|
| Неверная компания | Сверить с задачей пользователя |
| Неверная цена | Сверить с калькуляцией |
| Неверный НДС | Проверить ставку |
| Устаревший сертификат | Проверить файл и период действия |
| Неверный ГОСТ | Проверить источник |
| Данные заказчика из старого КП | Отметить как требующие проверки |
| Потеря результата | Сохранить итог в Google Drive |

## 7. Результат процесса

На выходе пользователь получает:

1. Текст коммерческого предложения.
2. Расчет цены.
3. Список использованных документов.
4. Список недостающих данных, если они есть.
5. Перечень приложений.
6. Рекомендации по проверке перед отправкой.
